Passo-a-passo Git

Para começar a usar o Git é preciso colocar informações em dois comandos:

git config --global user.name "nomedeusuario"
git config --global user.email "email@mail.com"

O comando git init serve para inicializar um novo repositório Git local na pasta atual.

O comando git status serve para ver o status das informações dentro do Git.

Untracked files significa que o arquivo ainda não foi trackeado para dentro do git.
Changes to be commited significa que o arquivo está preparado para ser commitado.

O comando git add 'nomedoarquivo' ou git add . serve para colocar o arquivo na área de preparação do Git - área de stage, deixando-o pronto para ser commitado.

O comando git commit -m "mensagem explicativa" serve para gravar as alterações colocadas na área de stage criando uma nova versão do projeto.

O comando git log serve para saber o histórico de versões existentes no projeto.

O HEAD serve para nos dizer qual a versão do projeto está sendo trabalhada no momento.
Para voltar a trabalhar em uma versão anterior do projeto utiliza-se o comando git checkout id-da-versao-a-ser-requerida.
O comando git log --all mostra todos os commits feitos.
O comando git checkout branch-principal faz voltar o HEAD para a última versão criada.
O comando git checkout nome-da-branch direciona o HEAD para tal branch.

Quando o terminal não volta a mostrar a área de inserção de comandos, basta teclar a seta pra baixo até aparecer na tela END, e então digitar Q.

O comando git checkout -b nome-da-branch serve tanto para criar uma nova branch diferente da principal como para direcionar o projeto para uma branch.

O comando git branch mostra as branchs existentes no projeto.

O comando git merge nome-da-branch adiciona o conteúdo da branch com o conteúdo da branch principal.

Primeiro, é preciso criar um novo repositório no GitHub.

Depois entra com o comando git remote add origin link-do-repo-github.

O comando git branch -M main renomeia a branch principal para ficar de acordo com a branch principal do GitHub.

Por fim, entra com o comando git push -u origin nome-da-branch-principal para 'empurrar' o repositório para o GitHub.

Para adicionar novas alterações basta commitar as alterações na máquina e enviar para o GitHub com o comando git push.

Para baixar repositórios do GitHub para a máquina

Utiliza-se o comando git clone url-do-projeto-no-github.

Para trabalhar em equipe.

Faz um fork no GitHub, clona o projeto na máquina, abre no VSCode com o comando code ., cria uma nova branch para trabalhar nela, faz as alterações necessárias no projeto, adiciona essas alterações na área de stage através do git add ., comita as alterações, envia/empurra para o GitHub através do git push origin nome-da-branch-criada e faz o pullrequest no GitHub.

Para fazer o pullrequest no GitHub clica em contribute e OpenPullRequest, e envia as alterações para serem aprovadas ou não pelo administrador do projeto.

Outros comandos

O comando git remote add origin url-github adiciona um repositório do GitHub à máquina.

O comando git remote -v lista os repositórios remotos adicionados à máquina.

O comando git pull origin main puxa as alterações do GitHub para a máquina.

Para adicionar um novo arquivo usa-se o comando echo > nome-do-arquivo.

Para mudar o nome da branch usa-se o comando git branch -m novo-nome-para-a-branch.

Para deletar uma branch usa-se git branch -d nome-da-branch-a-ser-deletada.

Para criar uma nova branch usa-se git branch nome-da-nova-branch.
